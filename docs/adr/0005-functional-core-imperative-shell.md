# Functional Core / Imperative Shell architecture

The Core is a pure functional layer — all functions take data in and return data out, with no I/O or side effects. The Integration (imperative shell) owns all I/O: reading Documents from disk or vault, resolving a Document's Type Declaration before calling Validation, injecting a Resolver for Wiki Link lookups, injecting a TypeRegistry for Referential Validation, and writing Scaffolding and Templating results back. This keeps the Core fully testable without mocks, runtime-agnostic, and embeddable in any JavaScript environment (Obsidian, browser, Node.js).
