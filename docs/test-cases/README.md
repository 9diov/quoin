# Test Cases

Concrete test cases for the Markdown Type System design.

These are specification fixtures, not executable tests. Each case defines:

- Type Definition Document schema
- Document frontmatter/body when relevant
- Validation Config and Integration lookup behavior when relevant
- Expected Validation Result shape

Use these cases to drive future unit tests for Core Validation, Parser behavior, and Integration lookup contracts.
